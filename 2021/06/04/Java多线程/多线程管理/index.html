Java之多线程管理

发表于 2021-06-04 | 分类于 Java |
字数统计: 2.4k

多线程管理

¶Lambda表达式

λ，希腊字母表中第是一个字母，英文名为Lambda

  1. 为了避免匿名内部类定义过多

  2. 可以让代码更加简洁

  3. 去掉无意义的代码，只留下核心的逻辑

  4. 其实质属于函数式编程的概念

¶函数式接口

Functional Interface（函数式接口），是Lambda表达式的关键

任何接口，如果只包含唯一一个抽象方法，那么它就是一个函数式接口

对于函数式接口，才能通过lambda表达式来创建该接口的对象

Lambda表达式：接口 对象 = (参数（接口中函数的参数）) -> {具体实现}

Lambda表达式只有一条语句的条件下才能省略{}和;，多行不可省略

¶静态代理模式

  1. 真实对象和代理对象需要实现同一个接口
  2. 代理对象要代理真实角色，需要将真实对象参数传递进去

意义：

  1. 代理对象可以做很多真实对象做不了的事
  2. 真实对象就专注做自己独有的事情

¶线程管理

¶五大状态：

  1. 创建状态：启动线程到就绪状态
  2. 就绪状态：获得CPU资源到运行状态
  3. 阻塞状态：解除阻塞状态到就绪状态
  4. 运行状态：释放CPU资源回到就绪状态，若休眠则到阻塞状态
  5. 死亡状态：运行状态执行完，外部干涉终止线程

¶线程方法

¶停止线程：

  1. 官方不推荐使用stop、destroy等方法停止【新版jdk已废弃】
  2. 建议使用一个标志位进行终止，如flag = false

¶线程休眠

  1. sleep(时间)指定当前线程阻塞的毫秒数
  2. sleep存在异常InterruptedException
  3. sleep时间达到后线程进入就绪状态
  4. slepp可以模拟网络延时，倒计时等
  5. 每一个对象都有一个锁，sleep不会释放锁

¶线程礼让

  1. 让当前正在执行的线程暂停，但不阻塞
  2. 将线程从运行状态转为就绪状态
  3. 让CPU重新调度。礼让不一定成功

¶线程强制执行

  1. Join合并线程，待此线程执行完成后再执行其他线程
  2. 期间其他线程阻塞，可以理解为插队

¶线程状态观测

Thread.State常量

一个线程可以在给定时间点处于一个状态

线程一旦进入死亡状态，就不能再次启动

这些状态是不反应任何操作系统线程状态的虚拟机状态

线程状态可以处于以下状态之一：

  1. NEW：尚未启用的线程状态
  2. RUNNABLE：Java虚拟机中执行的线程状态
  3. BLOCKED：被阻塞等待监视器锁定的线程状态
  4. TIMED_WAITING：等待另一个线程执行达到指定等到时间的线程状态
  5. TERMINATED：已退出的线程状态
