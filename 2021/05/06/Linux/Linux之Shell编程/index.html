¶Shell编程

¶Shell概述

  • 命令行解释器

  • 为用户提供向Linux内核发送请求来运行程序的界面系统级程序

  • 用户可以用Shell启动、挂起、停止甚至编写一些程序

  • 还是一个强大的编程语言

  • 易编写、易调试

  • 也是解释执行的脚本语言

  • Shell中可以直接调用Linux系统命令

  • Shell分类

    • Bourne Shell，主文件名为sh

      • 包括sh、ksh、Bash、psh、zsh
      • Bash和sh兼容
      • 目前主流Shell就是Bash

    • C Shell，主要在BSD版的Unix系统中，语法类似C

      • 包括csh、tcsh

    • 两种Shell不兼容

  • Linux支持的Shell

    • /etc/shells

¶Shell脚本执行方式

  • echo命令

    • echo 【选项】【输出内容】

    • -e，支持反斜线控制字符转换

      • \输出\本身
      • \a，警告音
      • \b，退格键，左删除键
      • \c，取消输出行末的换行符
      • \e，ESCAPE键
      • \f，换页符
      • \n，换行符
      • \r，回车键
      • \t，制表符，也就是tab键
      • \v，垂直制表符
      • \0nnn，八进制ASCII码输出，0为数字，nnn是八进制数
      • \xhh，按照十六进制ASCII码输出，hh是两位十六进制

  • 写脚本

    • vi XXX.sh

    • #! /bin/bash

      • 这不是注释，说明以下是shell脚本
      • 不写极容易脚本运行出错

  • 执行脚本

    • 赋予执行权限，直接运行

      • chmod 755 XXX.sh
      • ./XXX.sh

    • 通过Bash调用执行脚本

      • bash XXX.sh

  • dos2unix XXX.sh

    • 将脚本文件从Windows格式转换为unix格式
    • windows中编辑的脚本回车格式和linux回车不一致

¶Bash基本功能

  • 历史命令与命令补全

    • history，查看历史命令

      • -c，清空历史命令，谨慎使用

      • -w，历史命令写入保存文件~/.bash_history

      • 历史命令默认保存1000条

      • 环境变量文件/etc/profile中可修改

      • 调用

        • !n，重复执行第n条命令
        • !!，重复执行上一条命令
        • !字串，重复执行最后一条以该字串开头的命令

    • Tab键可以补全命令、文件或目录

    • 按两次Tab键给出所有选择浏览

  • 命令别名和常用快捷键

    • 命令别名

      • alias 别名 = ‘原命令’
      • alias，查询命令别名
      • 重启系统别名就会消失
      • 让别名永久生效 vi /root/.bashrc
      • 删除别名 unalias 别名

    • 命令执行时顺序

      • 第一顺位执行用绝对/相对路径执行的命令
      • 第二顺位执行别名
      • 第三顺位执行Bash的内部命令
      • 第四顺位执行按照$PATH环境变量定义的目录查找顺序找到的第一个命令

    • Bash常用快捷键

      • ctrl+a，光标移动到命令开头
      • ctrl+e，光标移动到命令末尾
      • （常用）ctrl+c，强制终止当前命令
      • （常用）ctrl+l，清屏
      • （常用）ctrl+u，删除或剪切光标之前的命令
      • ctrl+k，删除或剪切光标之后的命令
      • （常用）ctrl+y，粘贴剪切的内容
      • （常用）ctrl+r，在历史命令中搜索
      • （常用）ctrl+d，退出当前终端
      • ctrl+z，暂停，并放入后台
      • ctrl+s，暂停屏幕输出
      • ctrl+q，回复屏幕输出

  • 输入输出重定向

    • 标准输入输出

      • 键盘

        • 设备文件名/dev/stdin
        • 文件描述符0
        • 类型为标准输入

      • 显示器

        • /dev/stdout

          • 文件描述符为1
          • 标准输出

        • /dev/stderr

          • 文件描述符为2
          • 标准错误输出

    • 输出重定向

      • 作用：把命令执行结果保存下来

      • 标准输出重定向

        • 命令 > 文件

          • 以覆盖的方式
          • 命令的输出输出到指定文件或设备中

        • 命令 >> 文件

          • 以追加的方式
          • 输出到指定文件或设备中

      • 标准错误输出重定向

        • 错误命令 2> 文件

          • 以覆盖的方式
          • 把命令错误输出到指定文件或设备中

        • 错误命令 2>> 文件

          • 以追加方式
          • 把命令错误输出到指定文件或设备中

      • 正确和错误输出同时保存

        • 命令 > 文件 2>&1

          • 以覆盖的方式
          • 正确和错误输出保存到同一个文件中

        • 命令 >> 文件 2>&1

          • 以追加的方式
          • 正确和错误输出保存到同一个文件中

        • 命令 &> 文件

          • 以覆盖的方式
          • 正确和错误输出都保存到同一个文件中

        • 命令 &>> 文件

          • 以追加的方式
          • 正确和错误输出都保存到同一个文件中

        • 命令 >> 文件1 2>> 文件2

          • 正确的输出追加到文件1中
          • 错误的输出追加到文件2中

      • 命令 &> /dev/null，不保存任何结果，只执行

    • 输入重定向

      • wc 【选项】 【文件名】

        • -c，统计字节数
        • -w，统计单词数
        • -l，统计行数
        • ctrl+d，结束

      • 命令 < 文件，把文件作为命令输入

  • 多命令顺序执行与管道符

    • 多命令顺序执行

      • 分号；，多个命令按顺序执行

        • 中间命令错误也会继续执行

      • &&，逻辑与

        • 命令1正确执行后命令2才执行

      • ||，逻辑或

        • 命令1正确执行后命令2不会执行
        • 命令1不正确执行，命令2才执行

    • 管道符 |

      • 命令1 | 命令2
      • 命令1的正确输出作为命令2的操作对象

    • grep 【选项】 【文件或内容】

      • -i，忽略大小写
      • -n，输出行号
      • -v，反向查找
      • –color=auto，搜索出的关键字用颜色显示

  • 通配符与其他特殊符号

    • 通配符

      • ？，匹配一个任意字符

      • *，匹配0个或多个任意字符

      • 【】，匹配中括号中任意一个字符

        • 【abc】，代表一定匹配一个字符

      • 【-】，匹配中括号中任意一个字符，例【a-z】

      • 【^】，逻辑非，表示匹配不是中括号内的一个字符

    • 其他特殊符号

      • ‘ ’，单引号，无特殊含义
      • “ ”，除$、`和\以外无特殊含义
      • ，反引号，系统命令，Bash会先执行
      • $()，和反引号意义一致，建议使用，反引号容易看错
      • #，shell脚本中，写在开头的表示注释
      • $，调用变量的值
      • \，转义符，跟在\后面的特殊符号会变成普通字符

¶Bash变量

  • 变量概述

    • 计算机内存的单元
    • 其中存放的值可以改变，可以是数字、字母等
    • 用于保存暂时信息

  • 规则

    • 可以由字母、数字和下划线组成
    • 不能以数字开头
    • Bash中默认类型为字符串型
    • 变量和值用等号连接，含空格需要单引号括起来
    • 值可以使用\转义符
    • 如果需要增加变量的值，变量需要用双引号包含“$变量名”或者${变量名}包含
    • 若命令的结果作为值赋予变量，需用反引号或$()包含命令
    • 环境变量名建议大写，便于区分

  • 分类

    • 用户自定义变量（本地变量）

      • 只在当前的Shell中生效

      • 变量定义

        • a = 123

      • 变量叠加

        • a = "$a"456
        • a = ${a}789

      • 变量调用

        • echo $a

      • 变量查看

        • set

      • 变量删除

        • unset a

    • 环境变量

      • 概述

        • 主要保存系统操作环境相关的数据
        • 在当前Shell和所有子Shell当中生效
        • 若写入相应配置文件，环境变量就会在所有Shell中生效
        • pstree命令可以查询当前是否存在子shell

      • 设置

        • export 变量名 = 变量值，#申明变量
        • env，#查询环境变量
        • unset 变量名，#删除变量

      • 常见环境变量

        • PATH

          • 系统查找命令的路径的变量
          • echo $PATH可以查看
          • PATH = “$PATH” : /root/sh，叠加（临时）

        • PS1

          • 定义系统提示符的变量[root@localhost ~]#

          • env无法查询，set可以查

          • 详情

            • \d，显示日期，格式为“星期 月 日”
            • \h，简写主机名
            • \t，显示24时制时间，HH:MM:SS
            • \T，显示12时制时间，格式同\t
            • \A，显示24时制时间，格式为HH:MM
            • \u，显示当前用户名
            • \w，显示当前所在目录的完整名称
            • \W，显示当前所在目录的最后一个目录
            • #，执行的第几个命令
            • $，提示符，root用户为#，普通用户$

          • 举例

            • PS1 = ‘[\u@\t \w]$ ’
            • PS1 = '[\u@@ \h # \W]$ ’
            • PS1 = '[\u@\h \W]$ ’

    • 位置参数变量

      • 主要向脚本当中传递参数或数据，变量名和作用是固定的

      • 主要参数

        • $n

          • n为数字，$0代表命令本身
          • $1-$9代表1-9个参数，空格区分
          • 10以上需用{}，${10}

        • $*

          • 代表命令行中所有的参数
          • 把所有参数看为一个整体

        • $@

          • 也代表命令行中所有的参数
          • 但是把每个参数区分对待

        • $#

          • 这个变量代表命令行中所有参数的个数

    • 预定义变量

      • Bash中已经定义好的变量，变量名和作用是固定的

      • 主要参数

        • $?

          • 最后一次执行命令的返回状态
          • 为0表示上一个命令正确执行
          • 非0，值由命令决定，表示命令执行错误

        • $$

          • 当前进程的进程号（PID）

        • $!

          • 后台运行的最后一个进程的进程号

        • 补充

          • find /root -name xxx.sh &

            • 使用find命令查找xxx.sh文件并放入后台执行
            • &符号则为放入后台执行

          • read [选项] [变量名]

            • 键盘输入
            • -p “提示信息”，等待输入时显示
            • -t 秒数，指定等待输入时间
            • -n 字符数，只接受指定的字符数，才会执行
            • -s，隐藏输入的数据

¶Bash运算符

  • 数值运算与运算符

    • 数值运算

      • 方法1

        • declare +/-[选项] 变量名

          • 声明变量类型
          • -，给变量设定类型属性
          • +，取消变量的类型属性
          • -i，将变量声明为整数型
          • -x，将变量声明为环境变量
          • -p，显示指定变量的声明类型

        • 例

          • aa = 11
          • bb = 22
          • declare -i cc = $aa + $bb

      • 方法2

        • expr或let工具

          • aa = 11
          • bb = 22
          • cc = $(expr $aa + $bb)
          • 注:+号两侧必须有空格

      • 方法3

        • $((运算式))或$[运算式]

          • cc=$(($aa + $bb))
          • dd=$[aa + $bb]

    • 运算符

      • 优先级越大越优先
      • 优先级13 [+，- ]取正、负
      • 优先级12，[! , ~]，逻辑非、按位补码
      • 优先级11，[* ，/，%]，乘除，取模
      • 优先级10，[+ , -]，加减
      • 优先级9，[<< , >>]，按位左移，按位右移
      • 优先级8，[<= , >= , < , >]，大于小于，大于等于
      • 优先级7，[== ， != ]，等于，不等于
      • 优先级6，& 按位与
      • 优先级5，^ 按位异或
      • 优先级4，| 按位或
      • 优先级3，&&逻辑与
      • 优先级2，||逻辑或
      • 优先级1，[=，+=，-=， <<= 等]，赋值，运算赋值

  • 变量测试与内容替换

    • x = ${y - 新值}

      • y没设置，则x=新值
      • y为空，则x为空
      • y有值，则x=$y

    • x = ${y : 新值}

      • y没设置，则x=新值
      • y为空，则x=新值
      • y有值，则x=$y

    • x = ${y + 新值}

      • y没设置，则x为空
      • y为空，则x=新值
      • y有值，则x=新值

    • x = ${y :+ 新值}

      • y没设置，则x为空
      • y为空，则x为空
      • y有值，则x=新值

    • x = ${y = 新值}

      • y没设置，则x=新值，y=新值
      • y为空，则x为空，y不变
      • y有值，则x=$y，y不变

    • x = ${y := 新值}

      • y没设置，则x=新值，y=新值
      • y为空，则x=新值，y=新值
      • y有值，则x=$y，y不变

    • x = ${y ? 新值}

      • y没设置，则新值输出到标准错误输出（屏幕）
      • y为空，则x为空
      • y有值，则x=$y

    • x = ${y :? 新值}

      • y没设置，则新值输出到标准错误输出
      • y为空，则新值输出到标准错误输出
      • y有值，则x=$y

¶环境变量配置文件

  • 简介

    • source 配置文件或 . 配置文件(有空格)

      • 强制生效配置文件

    • 主要是对系统操作环境生效的系统默认环境变量

      • PATH
      • HISTSIZE
      • PS1
      • HOSTNAME

    • 环境配置文件

      • 1、/etc/profile

      • 2、~/.bash_profile，只对当前用户生效

      • 3、~/.bashrc，只对当前用户生效

      • 4、/etc/bashrc

      • 5、/etc/profile.d/*.sh

      • 读取顺序

        • 1234，然后读取命令提示符
        • 12345，然后读取具体脚本
        • 15，然后读取lang.sh，再读取/etc/sysconfig/i18n

  • 作用

    • /etc/profile

      • USER、LOGNAME、MAIL
      • PATH、HOSTNAME、HISTNAME
      • umask
      • 调用/etc/profile.d/*.sh文件

    • ~/.bash_profile

      • 调用~/.bashrc文件
      • 再PATH值后面加入家目录下的/bin这个目录

    • ~/.bashrc

      • 系统别名定义
      • 只针对当前用户

    • /etc/bashrc

      • PS1变量、PATH变量
      • umask
      • 调用/etc/profile.d/*.sh文件
      • 以上定义和调用针对不输入密码登录的情况

  • 其他配置文件和登录信息

    • 注销时生效，~/.bash_logout

    • 历史命令，~/bash_history

    • Shell本地登录信息，/etc/issue

      • \d，系统日期
      • \s，显示操作系统名称
      • \l，显示登录终端号
      • \m，显示硬件体系结构
      • \n，显示主机名
      • \o，显示域名
      • \r，显示内核版本
      • \t，显示当前系统时间
      • \u，显示当前登录用户序列号

    • 远程终端登录信息，/etc/issue.net

      • issue转义符不支持，只支持纯文本
      • 是否显示由/etc/ssh/sshd_config决定
      • 加入“Banner /etc/issue.net”行重启ssh服务才行

    • 登录欢迎信息，/etc/motd

      • 本地、远程登录都可以
      • 但是需要登录完成后才显示

¶正则表达式

  • 概念和比较

    • 用来在文件中匹配符合条件的字符串
    • 正则是包含匹配
    • grep、awk、sed等命令可以支持
    • 通配符用来匹配符合条件的文件名，完全匹配
    • ls、find、cp等命令不支持

  • 基础元字符

    • *，前一个字符匹配0或任意多次

      • grep “aa*” test.txt，匹配至少有一个a的行
      • grep “a*” test.txt，匹配所有行，包括空白行

    • . ，匹配除了换行符外任意一个字符

    • ，匹配行首，a就会匹配以a开头的行

    • $，匹配行尾，a$

    • []，匹配括号中任意一个字符，只匹配一个

    • [^]，匹配除括号中字符外任意一个字符

      • [a-z]，匹配不用小写字母开头的行
      • [a-zA-Z]，匹配不用字母开头的行

    • \，转义符

    • \ {n}，表示其前面字符恰好出现n次

      • [0-9]{4}，匹配4位数字
      • [1][3-8][0-9]{9}，匹配手机号

    • \ {n , }，表示其前面字符出现不小于n次

      • [0-9]{2,}，表示两位及以上的数字

    • \ {n, m}，表示其前面字符至少出现n次，最多m次

      • [a-z]{6,8}，匹配6到8位的小写字母

¶字符截取

  • cut

    • cut [选项] 文件名

      • -f 列号，提取第几列
      • -d 分隔符，指定分隔符分割列
      • cut -d “:” -f 1, 3 /etc/passwd，以:为间隔提取第1，3列
      • 不支持空格作为分隔符，不具体，分隔符必须具体

  • printf

    • printf ’输出类型 输出格式‘ 输出内容

      • 输出类型

        • %ns，输出字符串，n指几个字符
        • %ni，输出整数，n指几个数字
        • %m.nf，输出浮点数，分别指整数位数和小数位数，%8.2f，指8位数，2位小数，6位整数

      • 输出格式

        • \a，输出警告声音
        • \b，输出退格键
        • \f，清除屏幕
        • \n，换行
        • \r，回车
        • \t，水平输出退格键（TAB）
        • \v，垂直输出退格键（TAB）

      • printf后不能直接跟文件，或通过管道符跟文件

      • 只能printf ’%s‘ $(cat 123.txt)，这样输入

  • awk

    • 输出中支持print和printf

      • printf不会自动换行，print会，但Linux不含print

    • awk ’条件1 {动作1} 条件2{动作2}…’ 文件名

      • 条件

        • 一般使用关系表达式作为条件
        • x>10,x<=10等

      • 动作

        • 格式化输出
        • 流程控制语句

  • sed
